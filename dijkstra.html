Dijkstra's Algorithm
Dijkstra's Algorithm allows you to calculate the shortest path between one node (you pick which one) and every other node in the graph.

It is a greedy Algorithm.

Unlike in Prims, we do not consider the node-to-node distance but a cumulative distance keeping in mind the source node.
Algorithm
Dijkstra's Algorithm (G, w, s)
  1. INITIALIZE - SINGLE - SOURCE (G, s)
  2. S←∅
  3. Q←V [G]
  4. while Q ≠ ∅
  5. do u ← EXTRACT - MIN (Q)
  6. S ← S ∪ {u}
  7. for each vertex v ∈ Adj [u]
  8. do RELAX (u, v, w)
Time Complexity
The time complexity for Dijkstra's Algorithm is O(ElogV)


This can be improved by using Fibonacci Heaps which reduces the complexity to O(E+VlogV)
